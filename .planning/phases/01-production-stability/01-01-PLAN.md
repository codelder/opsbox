<objective>
Fix mutex poisoning DoS risk in agent HTTP handlers by migrating from std::sync::Mutex to tokio::sync::Mutex

Purpose: Prevent a single panic from poisoning the mutex and cascading to every subsequent HTTP request (DoS). The current code in agent/src/routes.rs lines 108 and 137 uses std::sync::Mutex with .lock().unwrap(), which panics on poisoned mutex -- a single panic in any handler permanently breaks the agent.

Output: Modified config.rs (tokio::sync::Mutex), routes.rs (async lock), Cargo.toml (parking_lot for future sync contexts)
</objective>
